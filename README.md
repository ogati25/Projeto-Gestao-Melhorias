# Projeto - Tickets de Bugs e Melhorias

Bem-vindo ao repositório de gerenciamento de tarefas do projeto!

Este repositório foi criado para organizar:

* Correções de bugs
* Melhorias no sistema
* Novas funcionalidades
* Ajustes visuais
* Sugestões de otimização
* Problemas encontrados durante testes

---

# Objetivo

Centralizar o acompanhamento das tarefas do projeto utilizando o sistema de *Issues* do GitHub.

Cada problema, melhoria ou sugestão deve ser registrada através de um ticket.

---

# Como abrir um ticket no GitHub

## 1. Acesse a aba "Issues"

No topo do repositório, clique em:

```txt
Issues
```

---

## 2. Clique em "New Issue"

Após entrar na aba de Issues:

* Clique no botão:

```txt
New Issue
```

---

## 3. Escolha o tipo de ticket

Crie o ticket conforme a necessidade:

* Bug
* Melhoria
* Nova funcionalidade
* Ajuste visual
* Documentação

---

# Modelo para BUG

Utilize este formato:

```md
## Descrição
Descreva o problema encontrado.

## Passos para reproduzir
1. Abrir o sistema
2. Clicar em X
3. Inserir Y
4. O erro acontece

## Resultado esperado
O que deveria acontecer.

## Resultado atual
O que realmente aconteceu.

## Prints ou vídeos
Anexe imagens se necessário.

## Ambiente
- Sistema Operacional:
- Navegador:
- Versão do projeto:
```

---

# Modelo para MELHORIA

Utilize este formato:

```md
## Melhoria proposta
Descreva a melhoria.

## Problema atual
Explique o que pode ser melhorado.

## Solução sugerida
Explique como poderia funcionar.

## Benefícios
Quais vantagens essa melhoria traz.
```

---

# Boas práticas

* Seja claro e objetivo
* Adicione imagens quando possível
* Informe exatamente onde ocorreu o problema
* Evite tickets duplicados
* Use títulos descritivos

---

# Exemplo de títulos

## Bug

```txt
Erro ao salvar usuário sem CPF
```

```txt
Tela de login trava após autenticação
```

## Melhoria

```txt
Adicionar filtro de pesquisa na tabela
```

```txt
Melhorar desempenho do carregamento inicial
```
